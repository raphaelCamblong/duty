duty is one binary. Install it, run `duty init`, and you have a task system —
plain markdown files in your repo, nothing else.

## Use it

```sh
duty init "My project"            # scaffolds ./duty
duty create track api             # a track is just a folder
duty create task "Ship the thing"
duty create task "Add auth" --in api   # target any board by path, from anywhere
duty get next                     # what should I work on?
duty status T-01 in-progress
duty status T-01 done
duty tui                          # watch it all live
```

That's it. Tasks are markdown files, tracks are folders, and a `BOARD.md` in
each track keeps the order. Everything is yours to grep, diff, and commit.

## The agent loop

duty was built to be driven by AI agents: commands are quiet, exit codes mean
things, and `--agent` turns any read into clean TSV. An agent's whole loop is
four calls:

```sh
duty get next --claim        # take the next actionable task
# ...work...
duty gates check --all       # tick the gates that pass
duty report --status done    # report and flip status in one write
```

Each recurring step is one atomic write, so running a swarm just works —
`get next --claim` hands each agent a distinct task under a tree-wide lock, and
their writes never collide. Hand your agent [the convention](/convention/) and
it knows the rules. Keep `duty tui` open and you see its progress the moment it
moves.
